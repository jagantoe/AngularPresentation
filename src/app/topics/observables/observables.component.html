Observables

Observables can be seen as an asynchronous stream of data. You can subscribe to an observables to receive any data it might emit over time. The observable can also throw errors and 'complete' to indicate that no more data will be emitted.

Subscribing

When subscribing to an observable we can pass an Observer object. This observer contains callbacks for each possible emission from an Observables.

  • next: called when the observable emits a new value
  • error: called when the observable encounters an error
  • complete: called when the observable completes
